## ✅ 驗證步驟 (按順序執行)

### 1️⃣ **CREATE - 新增問卷**

**步驟**:
```
1. 導航到 /admin 頁面
2. 在「問卷資訊」標籤中填寫:
   - 標題: "測試問卷 001"
   - 類型: 選擇任意類型
   - 說明: 填寫任意內容
   - 開始日期: 今天
   - 結束日期: 一週後
3. 在「問卷詳情」標籤新增至少 2 個題目:
   - 題目 1: "您的名字是？" (文字題)
   - 題目 2: "您滿意度如何？" (單選題，3 個選項)
4. 點擊「確認儲存」按鈕
```

**預期結果**:
- ✅ 頁面顯示 "問卷已成功儲存至 MySQL!" 提示
- ✅ 自動導航回 /surveys 列表頁面
- ✅ 新問卷 "測試問卷 001" **出現在列表頂部**
- ✅ 問卷狀態顯示為 "草稿"

**檢查 API 日誌**:
```
Request: POST /quiz/create
Response: {code: 200, message: "issue added successfully"}
```

```

   - 開始日期 (應該是今天)
   - 參與人數 (應該是 0，因為剛建立)
   - 狀態 (應該是 "草稿")
```

**預期結果**:
- ✅ 能看到列表中的所有問卷
- ✅ 每個問卷顯示的欄位完整準確

**預期結果**:
- ✅ 搜尋結果正確過濾

## 🐛 常見問題排查

### 問題 1: 新增問卷後列表不更新
**可能原因**:
- 後端 API 未返回 201 Created 狀態碼，或返回的 code ≠ 200
- fetchSurveys() 未被調用

**排查步驟**:
1. 開啟瀏覽器開發者工具 (F12)
2. 進入 Network 標籤
3. 創建新問卷並查看 POST /create 的響應
4. 檢查 response: {code: ?, message: "..."}
5. 如果 code ≠ 200，檢查後端日誌

### 問題 2: 編輯跳轉後 /admin 頁面顯示空白
**可能原因**:
- Router state 未正確傳遞問卷對象
- loadSurveyData() 未接收 preloadedSurvey 參數

**排查步驟**:
1. 檢查 confirmEdit() 是否調用了 router.navigate([], {state: {survey: ...}})
2. 檢查 survey-admin ngOnInit() 是否接收 state
3. 在 loadSurveyData() 加入 console.log 檢查是否接收到預加載數據

### 問題 3: 日期過濾無法正常工作
**可能原因**:
- dateStart/dateEnd 與 startDate/start_date 字段映射不符
- API 返回的日期格式不正確

**排查步驟**:
1. 打開瀏覽器控制台
2. 執行 `console.table(this.surveys)` (在 survey-list 組件中)
3. 查看實際返回的日期欄位名是什麼 (startDate? start_date?)
4. 檢查 onSearch() 中的日期比較邏輯是否兼容該欄位名

### 問題 4: 刪除操作不顯示 toast 提示
**可能原因**:
- triggerToast() 方法未正確實現或綁定
- 按鈕不在 component 的控制範圍內

**排查步驟**:
1. 檢查 triggerToast() 方法是否存在且可呼叫
2. 在 confirmDelete() 中加入 console.log 確認代碼已執行
3. 檢查 toast UI 元素是否在模板中定義
